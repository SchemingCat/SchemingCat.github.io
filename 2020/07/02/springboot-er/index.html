SpringBoot的Starter

SpringBoot为了简化配置，提供了很多的Starter。它打包了与常用模块相关的JAR包，并完成了自动配置。然后组装成了Starter，在开发时只需引入Starter即可正常使用功能

Starter 说明
string-boot-starter-web 用于构建Web，包含Rest风格框架，SpringMVC和默认的嵌入式容器Tomcat
string-boot-starter-test 用于测试
string-boot-starter-jpa 带有Hibernate的Spring Date JPA
string-boot-starter-jdbc 传统的jdbc，轻量级框架可用。学习成本低，但最好使用JPA或Mybatis
string-boot-starter-thymeleaf 支持Thymeleaf模板
string-boot-starter-mail 支持Java Mail，Spring Email邮件
string-boot-starter-integration Spring框架创建的一个API，面向企业应用集成(EAI)
string-boot-starter-mobile Spring MVC扩展。用来简化手机上的Web应用程序开发
string-boot-starter-data-redis 通过Spring Data Redis、Redis Client使用Redis
string-boot-starter-validation Bean Validation是一个数据验证的规范，Hibernate Validator是一个数据验证框架
string-boot-starter-websockices 相对非持久的协议HTTP，Websocker是一个持久的协议
string-boot-starter-web-services SOAP Web Services
string-boot-starter-hateoas 为服务添加HATEOAS功能
string-boot-starter-security 用Spring Security进行身份验证和授权
string-boot-starter-data-rest 用Spring Data REST 公布简单的REST服务

视图技术

SpringBoot 主要支持Thymeleaf、Freemarker、Mustache、Groovy Templates等模板

Thymeleaf

Thymeleaf的基本使用：

<p th:text="${message}?:'text'">内容</p>

这个语法表示接收从控制层传递过来的名为message的值并替换掉原有的内容。如果没有传过来，则使用text作为p标签的内容。

使用Thyeleaf

加入Thymeleaf依赖

<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-thymeleaf</artifactId>
</dependency>

在模板文件中加入解析

在HTML文件中加入<html lang=”en” xmlns:th=”http://www.thymeleaf.org"\>

配置视图解析器

SpringBoot默认的页面映射路径为:classpath:/templates/*.html;静态文件路径为:classpath:/static/

在配置文件(properties或yml)中进行配置

spring
  thymeleaf
    mode: HTML5 # 表示Thymeleaf模式为HTML5
    encoding: UTF-8 # 表示Thymeleaf编码格式为UTF-8
    content-type: text/html # 文档类型为text/html
    cache=false # 为了便于测试，在开发过程中将关闭缓存

注意：由于Thymeleaf检查HTML格式严格。如果HTML格式不对，则会报错。如果想禁止这种严格的语法检查模式，可以在配置文件中加入spring.thymeleaf.mode=LEGACYHTML5来解决。

Thymeleaf基础语法

常用th标签

  1. th:text
     <div th:text="${message}"></div>

    用于显示控制器传入的值
    如果message不存在，需要显示默认值可用以下的方式

     <div th:text="${message}?:'默认值'"></div>
  2. th:object
     <div th:object="${user}"></div>

    用于接收后台传递过来的对象

  3. th:action
     <form th:action="@{/article/}+${article.id}" method="post"></form>

    用于指定表单提交地址

  4. th:value
     <input type="text" th:value="${user.name}" id="name"/>

    将对象的值替换成value的值

  5. th:field
     <input type="text" th:field="${user.name}" id="name"/>

    用来绑定后台对象和表单数据

URL写法

Thymeleaf是通过语法@{…}来处理URL的。需要使用”th:href”和”th:src”等属性。

<a th:href="@{/}">相对路径</a>
<a th:href="@{http://text.com}">绝对路径</a>
<a th:href="@{css/jquery.js}">默认访问static下的css文件夹内的jquery.js文件</a>

注意：在通常url路径中，如果要传入多个参数用’&’分隔，但在Thymeleaf中，使用’,’分隔多个参数

条件判断

  1. th:if
     <a th:href="@{css/js}" th:if="{user.id == null}"></a>

    当条件满足时，a标签才显示

    if的表达式

    • gt
      • 大于
    • ge
      • 大于或等于
    • eq
      • 等于
    • lt
      • 小于
    • le
      • 小于或等于
    • ne
      • 不等于
  2. th:unless
     <a th:href="@{css/js}" th:unless="{user.id == null}"></a>

    当条件不成立时，a标签才显示

  3. th:switch
     <div th:switch="{user.name}">
   <p th:case="administrator">administrator</p>
   <p th:case="user">user</p>
   <p th:case="*">无匹配</p>
 </div>

    当user的name等于administrator时，显示administrator标签。如果所有的值都不满足，则显示’无匹配’。

字符串替换

有时需要对文字中的某一处地方进行替换。可以通过字符串拼接操作完成。

<span th:text="欢迎你登录，'+${name}+',管理员"></span>

或者

<span th:text="欢迎你登录，${name},管理员"></span>

第二种形式限制较多，只能包含${}表达式，不能包含其他常量，条件表达式等。

运算符

算数运算符

  <span th:text="1+1"></span>

表示将1+1的结果放入span标签内

公用对象

Thymeleaf提供了一系列的公用对象，通过’#’访问

  • 格式化时间
      <td th:text="${#dates.format(item.createTime,'yyyy-MM-dd HH:mm:ss')}">格式化时间</td>
  • 判断是不是空字符
      <td th:if="${#strings.isEmpty(name)}">空的</td>
  • 是否包含(区分大小写)
      <td th:if="${#strings.contains(name,'张三')}">包含张三</td>

遍历循环

遍历对象
    <ul th:each="User:${Users}">
      <li th:text="${User.id}}"></li>
      <li th:text="${User.name}}"></li>
    </ul>

可以通过’th:each=”Object:${Object}”‘标签来遍历。

遍历分页
    <ul th:each="item:${page.content}">
      <li th:text="item.id">id</li>
      <li th:text="item.title">title</li>
    </ul>

可以通过’th:each=”item:${page.content}”‘标签来处理page对象。

遍历列表

使用’th:each=”item:${list}”‘
如果list只有一个元素，则用

    <div th:each="item:${list}">
      <li th:text="${item}"></li>
    </div>

如果list有多个元素，用

    <div th:each="item:${list}">
      <li th:text="${item.id}"></li>
      <li th:text="${item.name}"></li>
    </div>
遍历数组
    <div th:each="item:${arrays}">
      <li th:text="item"></li>
    </div>

使用’th:each=”item:${arrays}”‘遍历数组

遍历集合
    //遍历key
    <div th:each="item:${map}">
      <li th:text="item.key"></li>
    </div>

    //遍历value
    <div th:each="item:${map}">
      <li th:text="item.value"></li>
    </div>

    //遍历key-value
    <div th:each="item:${map}">
      <li th:text="item"></li>
    </div>

通过’th:text=”${item.key}”‘显示集合的key，通过’th:text=”${item.value}”‘显示集合的value

创建公共代码块

当一个项目的头部(header)和尾部(footer)需要重复显示时，可以创建一个公共块，然后在需要重复使用的地方调用

创建重复代码块

可以使用th:fragment=”header”标签来标记重复代码块

<div th:fragment="footer">
  重复的头部(header)
</div>
调用重复代码块

使用th:include或th:replace标签调用。标签根据fragment值来调整

<div th:replace="~{common :: header}"></div>
<div th:include="~{common :: footer}"></div>

th:include表示将当前标签替换为模板中的标签，th:replace表示只在当前标签内加载模板中的内容

分页处理

  1. 控制器传入Page对象
     Pageable pageable = PageRequest.of(start, limit, sort);
 Page<User> page = userMapper.findAll(pageable);
 ModelAndView modelAndView = new ModelAndView();
 modelAndView.addObject("pages",page);
 return modelAndView;
  2. 接收Page对象并处理
     <a th:href="@{/article(start=0)}">首页</a>
 <a th:if="${not page.isLast()}" th:href="@{/article(start=${page.number+1})}">下一页</a>
 <a th:if="${not page.isFirst()}" th:href="@{/article(start=${page.number-1})}">上一页</a>
 <a th:href="@{/article(start=${page.totalPages-1})}">尾页</a>

    th:if=”${not page.isFirst()}”表示判断当前页数是否为第一页。在Thymeleaf中可以直接调用对象的方法

实体类注入属性

在需要注入的类上面加上注解：@Component和@ConfigurationProperties(prefix = “name”)

@Component  //等于创建了<bean id="user' class="cn.jbit.entity.User"/>。可以通过@Resources实现自动注入
@ConfigurationProperties(prefix = "user")
public class User{
  private String name;
  private int age;
}

在配置文件中根据prefix的名称注入属性

user.name=张三
user.age=18

需要注意：如果注入中文，显示会报错。需要修改字符编码：设置-Editor-File Encodings
