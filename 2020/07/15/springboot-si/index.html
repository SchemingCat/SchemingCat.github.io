整合Spring Data JPA

Hibernate是一个ORM框架，而JPA则是一种ORM规范。JPA和Hibernate的关系就像JDBC与JDBC驱动的关系，即JPA制定了ORM规范。而Hibernate是这些规范的实现(事实上,是先有Hibernate后有JPA, JPA规范的起草者也是Hibernate的作者) 。因此从功能上来说, JPA相当于Hibernate的一个子集。

使用JPA的好处

使用JPA可以根据实体类来在数据库中创建对应的表。可以节约开发时间。

  1. 创建数据库
     create database 'JPAtest' default character set utf8
  2. 配置文件配置
     spring.datasource.type=com.alibaba.druid.pool.DruidDataSource
 spring.datasource.url=jdbc:mysql///JPAtest
 spring.datasource.username=root
 spring.datasource.password=root
 spring.jpa.show-sql=true
 spring.jpa.database=mysql
 spring.jpa.hibernate.ddl-auth=update
 spring.jpa.properties.hibernate.dialect=org.hibernate.dialect.MySQL57Dialect

    spring.jpa.show-sql：是否打印JPA执行时生成的sql
    spring.jpa.database：JPA对应的数据库
    spring.jpa.hibernate.ddl-auth：在项目启动时根据实体类更新数据库中的表(其他可选有：create、create-drop、validate、no)
    spring.jpa.properties.hibernate.dialect：使用的数据库方言是MySQL57Dialect

  3. 创建实体类
     @Entity(name = "user")
 public class User{
     @Id
     @GeneratedView(strategy = GenerationType.IDENTITY)
     private Integer id;
     @Column(name = "userName", nullable = false)
     private String userName;
     @Column(name = "userPassWord", nullable = false)
     private String userPassWord;
     @Column(name = "age", nullable = false)
     private Integer age;
     @Transient
     private String Data;
 }
    • @Entity
      • 表示该类是一个实体类。在项目启动时会根据实体类创建一张表。表的名称为name的值。如果没有name属性，默认用类名作为表名
    • @Id
      • 表示该属性为一个主键。
    • @GeneratedView
      • 表示主键自动生成，strategy表示主键的生成策略
    • @Column
      • 表示该属性生成到表中的列。默认用属性名作为列名，使用name属性后name的值为列名。nullable表示是否非空
    • @Transient
      • 表示创建表时忽略该属性

配置数据源

创建类配置数据源。根据配置文件中的配置生成两个数据源

@Configuration
public class DataSourceConfig{
    @Bean
    @ConfigurationProperties("spring.datasource.one")
    DataSource dsOne(){
        return DruidDataSourceBuilder.create().build();
    }

    @Bean
    @ConfigurationProperties("spring.datasource.two")
    DataSource dsTwo(){
        return DruidDataSourceBuilder.create().build();
    }
}

配置JdbcTemplate

当开发者没有提供JdbcTemplate时，SpringBoot会自己提供一个JdbcTemplate实例。当配置多数据源时，需要由开发者提供JdbcTemplate实例

@Configuration
public class JdbcTemplateConfig{
    @Bean
    JdbcTemplate jdbcTemplateOne(@Qualifier("dsOne")DataSource dataSource){
        return new JdbcTemplate(dataSource);
    }

    @Bean
    JdbcTemplate jdbcTemplateTow(@Qualifier("dsTow")DataSource dataSource){
        return new JdbcTemplate(dataSource);
    }
}

创建Mybatis配置

主要是提供SqlSessionFactory实例和SqlSessionTemplate实例

@Configuration
@MapperScan(value = "org.sang.mapper1", sqlSessionFactoryRef = "sqlSessionFactoryBean1")
public class MyBatisConfigOne{
    @Resource(name = "dsOne")
    DataSource dsOne
    @Bean
    SqlSessionFactory sqlSessionFactory1() throws Exception{
        SqlSessionFactoryBean factoryBean = new SqlSessionFactoryBean();
        factoryBean.setDataSource(dsOne);
        return factoryBean.getObject();
    }

    @Bean
    SqlSessionTemplate sqlSessionTemplate1() throws Exception {
        return new SqlSessionTemplate(sqlSessionFactoryBean1());
    }
}

多数据源需要配置多个MyBatisConfigOne。注意方法不能同名

NoSql

NoSQL是指非关系型数据库，非关系型数据库和关系型数据库两者存在许多显著的不同点。其中最重要的是NoSQL不使用SQL作为查询语言，其数据存储可以不需要固定的表格模式。一般都有水平可扩展性的特征。
NoSQL主要有如下几种不同的分类:

  • Key/Value键值存储。这种数据存储通常都是无数据结构的,一般被当作字符串或者二进制数据,但是数据加载速度快,典型的使用场景是处理高并发或者用于日志系统等,这一类的数据库有Redis. Tokyo Cabinet等。
  • 列存储数据库。列存储数据库功能相对局限,但是查找速度快,容易进行分布式扩展,一般用于分布式文件系统中,这一类的数据库有HBase, Cassandra等。
  • 文档型数据库。和Key/Value键值存储类似,文档型数据库也没有严格的数据格式,这既是缺点也是优势,因为不需要预先创建表结构,数据格式更加灵活,一般可用在Web应用中,这一类数据库有MongoDB, CouchDB等。
  • 图形数据库。图形数据库专注于构建关系图谱,例如社交网络,推荐系统等,这一类的数据库有Neo4J、 DEX等

配置Redis

在配置文件中配置连接信息

spring.redis.database=0 #使用Redis的第几个库。一共有0-15个库
spring.redis.host=127.0.0.1 #Redis的实例地址。如果使用的本地的Redis写127.0.0.1即可，如果Redis部署在其他服务器上，则写服务器的IP地址
spring.redis.port=6379 #Redis端口。默认6379
spring.redis.password=administrator #Redis登录密码
spring.redis.jedis.pool.max-active=8    #Redis最大连接池数量
spring.redis.jedis.pool.max-idle=8  #Redis最大空闲连接池数量
spring.redis.jedis.pool.max-wait=-1ms   #最大阻塞时间。默认为-1，表示没有限制
spring.redis.jedis.pool.min-idle=0  #最小连接池空闲数量

如果项目使用了Lettuce，将jedis更改为lettuce即可

使用Redis

存入String类型

@Resource
StringRedisTemplate stringRedisTemplate;

public void test(){
    ValueOperations<String, String> ops = stringRedisTemplate.opsForValue();
    //存入Redis
    ops.set("userName","张三");

Redis集群

实际开发中，为了提高Redis的扩展性。一般搭建Redis集群

Redis集群管理工具redis-trib.rb依赖Ruby环境。首先需要安装Ruby环境

在redis的根目录下(和src同级)创建redisCluster文件夹。将src目录下的redis-trib.rb复制到redisCluster目录下

在redisCluster目录下根据启动的端口和ip不同创建不同的文件夹，并将redis.conf文件往创建的文件夹中分别复制一份。然后对目录中的redis.conf文件进行修改

修改127.0.0.1:8001目录下的redis.conf文件

port 8001
#bind 127.0.0.1
cluster-enabled yes #表示开启集群
cluster-config-file nodes-8001.conf #表示集群节点的配置文件
protected no
daemonize yes
requirepass administratot   #redis密码
masterauth

全部修改完后进入redis根目录启动

src/redis-server redisCluster/8001/redis.conf
src/redis-server redisCluster/8002/redis.conf
src/redis-server redisCluster/8003/redis.conf
src/redis-server redisCluster/8004/redis.conf

当Redis都启动成功后进入redisCluster目录下对redis-trib.rb文件进行修改。首先需要配置密码。在文件内搜索到如下一行

@r = Redis.new(:host => @info{:host}, :port => @info{:port}, :timeout => 60)

修改这一行，改为

@r = Redis.new(:host => @info{:host}, :port => @info{:port}, :timeout => 60, :password => 'administrator')

创建集群

在根目录执行如下命令创建Redis集群

redisCluster/redis-trib.rb create --replicas 1 192.168.179.129:8001 192.168.179.129:8002 192.168.179.129:8003 192.168.179.129:8004

192.168.179.129为Redis所在主机的IP地址。replicas表示每个主节点的slave数量。在集群的创建过程中会分配主机和从机。每个集群在创建过程中都将分配到一个唯一的id并分配到一段slot

当集群创建成功后，进入redis根目录登录任意redis实例。

src/redis-cli -p 8001 -a administrator -c

-p表示要登录的集群端口，-a表示登录的密码，-c表示以集群的方式登录。登录成功后可以通过cluster info命令查询集群状态信息。通过cluser nodes命令可以查询集群节点信息

添加主节点

在启动成功后可以使用add-node将后续的节点添加到集群中，如

redisCluster/redis-trib.rb add-node 192.168.179.129:8005 192.168.179.129:8001

由于slot已经被之前创建的实例分配完了，新加入的没有slot，也就是说新添加的没有存储数据的机会。这时候需要从其他实例中拿出一部分分配给新实例

redisCluster/redis-trib.rb reshard 192.168.179.129:8001

执行命令中，三个核心配置需要手动配置

  • 第一个配置是要拿出多少个slot分配给新实例,这里配置了1000个。
  • 第二个是把拿出来的1000个slot分配给谁,输入接收这1000个slot的Redis实例的id,这个id在节点添加成功后就可以看到,也可以进入集群控制台后利用cluster nodes命令查看。
  • 第三个配置是这1000个slot由哪个实例出,例如从端口为8001的实例中拿出1000个slot分配给端口为8007的实例,那么这里输入8001的id后按回车键,再输入done按回车键即可,如果想将1000个slot均摊到原有的所有实例中,那么这里输入all按回车键即可。

添加从节点

redisCluster/redis-trib.rb add-node --master-id ? 192.168.179.129:8006 192.168.179.129:8001

–master-id表示该从节点的id。第一个ip地址表示从节点的地址，第二个ip地址表示集群中任意一个实例的地址

删除节点

redisCluster/redis-trib.rb del-node 192.168.179.129:8006 ?

ip地址表示为集群中任意一个实例，最后的参数表示要删除的节点id。如果删除的节点有slot，则删除失败。需要先将slot先分配出去。

配置Redis

创建RedisConfig，完成对Redis的配置

@Configuration
@ConfigurationProperties("spring.redis.cluster")
public class RedisConfig{
    List<Integer> ports;
    Strign host;
    JedisPoolConfig poolConfig;

    @Bean
    RedisClusterConfiguration redisClustErConfiguration(){
        RedisClusterConfiguration configuration = new RedisClusterConfiguration();
        List<RedisNode> nodes = new ArratList<>();
        for(Integer port : ports){
            nodes.add(new RedisNode(host, port));
        }
        configuration.setPassword(RedisPassword.of("administrator"));
        configuration.setClusterNodes(nodes);
        return configuration;
    }

    @Bean
    JedisConnectionFactory jedisConnectionFactory() {
        JedisConnectionFactory factory = new JedisConnectionFactory(redisClusterConfiguration(), poolConfig);
        return factory;
    }
}
