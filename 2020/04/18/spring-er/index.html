什么是AOP

AOP是面向切面(Aspect Orinted Programming)编程。是软件编程思想发展到一定程度的产物。是面向对象编程的补充。一般适用于具有横切逻辑的场合，如访问控制，事务管理，性能监控

AOP的特点

能够在不改变原代码的基础上织入新的功能

AOP的使用

xml配置

<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
    xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
    xmlns:aop="http://www.springframework.org/schema/aop"
    xsi:schemaLocation="http://www.springframework.org/schema/beans 
        https://www.springframework.org/schema/beans/spring-beans.xsd 
        http://www.springframweork.org/schema/aop 
        http://www.springframework.org/schema/aop/spring-aop.xsd">
  <aop:aspectj-autoproxy proxy-target-class="true"/>
</beans>

和AOP相关的配置都放在<aop:cnofig>标签中

<aop:cnofig>
  <aop:pointcut id="" expression="execution(public void setUser(cn.jbit.entity.User))"/>
</aop:cnofig>

配置切入点的标签<aop:pointcut>的expression属性可以配置切入点表达式，用来织入与表达式匹配的方法

expression="expression(public void setUser(User user))"

expression：切入点标识符，括号中是一个切入点表达式。常用的模糊查询表达式如下：
public * setUser(User user)：匹配所有类型的返回值

public void *(User user)：匹配所有的方法名

public void setUser(..)：匹配所有的参数个数和类型

* com.service.*.*(..)：匹配com.service包下的所有类和方法

* com.service..*.*(..)：匹配com.service包和子包下的所有类和方法

将先执行的方法a和后执行的方法b织入

<bean id="testa" class="方法a和方法b所在的类路径"/>
<aop:config>
  <aop:pointcut id="pointcutTest/切入点id" expression="execution(public void setUser(cn.jbit.entity.User))"/>
  <aop:aspect ref="testa">
    <!--将方法a作为前置增强并引用需要增强的方法切入点id-->
    <aop:before method="a" pointcut-ref="pointcutTest"/>
    <!--将方法b作为后置增强并引用需要增强的方法切入点id-->
    <aop:after-returning method="b" pointcut-ref="pointcutTest" returning="result"/>
  </aop:aspect>
</aop:config>

AOP常用的增强方式

异常抛出增强

特点是在目标抛出异常时，织入增强处理。可为其他异常模块提供统一，可插拔的异常处理方案
包含增强方法的JavaBean

pubic class test1{
  public void afterThrowing(JoinPoint jp,RuntimeException e){
    system.out.print(jp.getSignature().getName()+"方法异常："+e);
  }
}

xml配置如下

<!--声明增强方法所在的bean-->
<bean id="theLogger" class="test1路径"/>
<!--配置切面-->
<aop:config>
  <!--定义切入点(需要增强的方法所在)-->
  <aop:pointcut id="" expression="execution(* server.UserServer.*(..))"/>
  <!--引用增强方法-->
  <aop:aspect ref="增强方法的bean">
    <aop:after-throwing method="增强方法名" pointcut-ref="切入点id" throwing="e"/>
  </aop:aspect>
</aop:config>

注意：op:after-throwing标签即为异常增强

最终增强

最终增强的特点是无论方法是抛出异常还是正常退出，该增强都会执行。类似finally语句块，一般用于释放资源

语法和异常抛出增强基本一致，只需要更改<aop:aspect>标签内的子标签即可

<aop:after method="增强方法名" pointcut-ref="切入点id"/>

环绕增强

环绕增强在目标的前后都可以执行织入增强处理。在环绕增强中，可以获取和修改传递的参数，返回值。可以对他进行异常处理，甚至可以决定方法是否被执行

public class test1{
  public void testa(ProceedingJoinPoint jp) throws Throwable{
    system.out.print("方法执行前");
    try{
      Object result = jp.proceed();//执行方法并获得返回值
      return result;
    }catch(Throwable e){
      system.out.print("方法异常："+e);
      throw e;
    }finally{
      system.out.print("方法执行结束");
    }
  }
}

xml配置

<aop:around method="增强方法名" pointcut-ref="切入点id"/>

使用注解进行操作

实现控制反转

开启注解扫描：

<context:component-scan base-package="cn.jbit"/>

自动扫描带有@Service等注解的文件配置成bean

在需要定义bean的类名上面加上注解即可创建对应的bean。效果等同于：<bean id=”名称” class=”路径”/>
注解：

  • @Repository(“名称”)：用于标注dao类(XXXDao.java)
  • @Service(“名称”)：用于标注业务类(XXXServerImpl.java)
  • @Controller(“名称”)：用于标注控制器类
  • @Component(“名称”)：用于标注控制器类(XXXDaoImpl.java)

完成属性的装配

使用：在需要配置的属性名上面加入注解即可

@Autowired

该注解自动为属性注入所依赖内容。此时可以省略属性的set方法

如果有一个以上的类型自动匹配时，则可以使用注解指定需要的bean名称

例：

@Autowired
@Qualifier("bean名称")
private UserDao userDao;

也可以直接指定某个bean

@Rrsource("")

使用注解配置的xml

使用注解即可为指定特定的名称的bean注入属性

@Resource(name="name")

如果没有指定name，只是@Resource。则会自动匹配与属性名相同的bean注入

配置切面

配置xml文件

<bean class="切面所在的路径"/>
<aop:aspectj-autoproxy/>
  1. 使用@Aspect注解在类上添加切面
  2. 使用@Before(“execution(* cn.jbit.dao..*(..))”)将方法定义为切面增强方法
  3. @Before为前值增强，@AfterReturning(pointcut = “execution(* cn.jbit.dao..*(..))”, returning = “returnValue”)为后置增强。returning为方法接收的返回值名称。如String a则为returning = “a”

    如果多个方法需要匹配同一个表达式，则可以简化。

如前置增强和后置增强都需要匹配dao包和其子包(execution(* cn.jbit.dao..*(..))，则可以

@Pointcut("execution(* cn.jbit.dao..*(..)")
public void test(){}

@Before(" test()")
public void before(){
  //前值增强
}

@AfterReturning(pointcut = "test()", returning = "a")
public void aterReturning(Object a){
  //后置增强
}

配置增强类型

异常增强：

@AfterThrowing(pointcut = "test()" , throwing = "e")
public void afterThrowing(RuntimeException e){
  //异常增强
}

最终增强：

@After("test()")

环绕增强：

@Around("test()")
public void around(ProceedingJoinPoint jp){
  //调用方法：jp,getSignature().getName();
  //传入参数：Arrays.toString(jp.getArgs);
  Object obj = jp.proceed();//执行方法并返回返回值
}
