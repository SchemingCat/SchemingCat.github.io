Spring是什么

Spring是分层的Java SE/EE应用，full-stack（全栈式）轻量级开源框架。以IOC和AOP为内核

Spring两大核心

Spring的两大核心分别是IOC和AOP

  • IOC
    • Inverse Of Control/反转控制/控制反转
    • 降低程序间的耦合
  • AOP
    • Aspect Oriented Programming/面向切面编程
    • 在不改变原有代码的基础上织入新的功能

Spring的优势

轻量级，开源，方便解耦，声明式（配置）事务的支持，AOP的支持，方便测试，方便集成其他框架

程序的耦合和解耦

什么是耦合

简单的说，程序之间的依赖叫做耦合

  1. 类之间的依赖
  2. 方法之间的依赖

什么是解耦

降低程序之间的依赖/降低耦合；叫做解耦

开发中应做到：编译时不依赖，运行时才依赖

IOC的概念和作用

概念

什么是IOC？为什么叫控制反转？

IOC是通过Bean工厂来实例化对象。由之前自己创建对象到将创建对象的权利交给工厂或框架来做，这就叫做控制反转

IOC的作用

削减计算机的耦合，但不能彻底消除耦合

为什么使用IOC

通常的实例化对象如果出现类名的变动，由于类之间的高耦合。需要将实例化全部修改一遍，而用了IOC解耦后只需要修改配置文件即可

基于xml的IOC环境搭建

首先获取Spring的常用jar包(可去maven官网下载，或使用maven项目)

  • commons-logging
  • mysql-connector
  • spring-aop
  • spring-beans
  • spring-context
  • spring-core
  • spring-expression
  • spring-jcl

    配置applicationContext.xml文件

<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
    xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
    xsi:schemaLocation="http://www.springframework.org/schema/beans
        https://www.springframework.org/schema/beans/spring-beans.xsd">
    <!-- 将实例化对象交给Spring -->
    <bean id="UserDaoImpl" class="cn.jbit.news.dao.Impl.UserDaoImpl"></bean>
    <bean id="UserServetImpl" class="cn.jbit.news.servet.impl.UserServetImpl"></bean>
</beans>

创建对应的bean，可以通过对应的方法创建对象。这样就降低了程序之间的耦合

创建对象

立即加载

创建对象是采用立即加载，读取完配置文件所有对象就已经被创建

ApplicationContext ac = new ClassPathXmlApplicationContext("applicationContext.xml/文件位置");
IUserDao userDao = (IUserDao)ac.getBean("xml配置的id");

提示：(IUserDao)ac.getBean(“xml配置的id”);和ac.getBean(“xml配置的id”,IUserDao.class);是一样的

延迟加载

创建对象是采用延迟加载，等实例化时才创建对象

Resource resource = new ClassPathResource("bean.xml");
BeanFactory factory = new XmlBeanFactory(resource);

IUserDao userDao = (IUserDao)factory.getBean("UserDaoImpl");

Spring对Bean的管理

  1. 采用默认构造函数

    在Spring配置文件中使用bean标签，配上id和class属性后，且没有其他属性和标签时，采用的就是默认构造函数创建bean对象。如这时候类中无默认构造函数则对象无法创建

     <bean id="UserDaoImpl" class="cn.jbit.news.dao.Impl.UserDaoImpl"></bean>

  2. 使用普通工厂中的方法创建对象（使用某个类中的方法创建对象，并存入Spring容器）

    例：A工厂有一个方法返回了B对象，如果这时候想获取B对象，但是由于A工厂没有默认构造函数。所以只能采用第二张方法来创建bean对象

     <bean id=”工厂名称” class=”工厂全限定类名”></bean>
 <bean id=”B对象名称” factory-bean=”工厂名称/id” factory-method=”工厂里返回B对象的方法名”></bean>

  3. 使用工厂中的静态方法创建对象（使用某个类中的静态方法创建对象，并存入Spring容器）

     <bean id=”B对象名称” class=”工厂全限定类名” factory-method=”工厂里返回B对象的方法名”></bean>

bean对象的作用范围调整

bean标签的scope属性

作用：指定bean的作用范围;取值：(常用的是单例和多例)

  • singleton：
    • 单例（默认值）
  • prototype：
    • 多例
  • request：
    • 作用于web应用的请求范围
  • session：
    • 作用于web应用的会话范围

bean的生命周期

单例对象

出生：容器被创建时

活着：只要容器在，对象一直活着

死亡：容器销毁

多例对象

出生：对象被使用时

活着：对象在使用过程中一直活着

死亡：对象长时间不使用，且没有被别的对象引用时，由java的垃圾回收器回收

依赖注入

常用的注入方式

能注入的数据有三类:1.基本类型和String;2.其他bean类型(在配置文件中或注解配置过的bean);3.负责类型/集合类型

注入的方式

  1. 使用构造函数
  2. 使用set方法（常用）
  3. 使用注解
  4. p命名空间注入

构造函数注入

  1. 在类中定义可注入的变量
  2. 创建有参的构造器

    经常变化的数据不适合注入

标签：constructor-arg

位置：bean内部

属性：

  • type：
    • 用于指定要注入的数据类型，该类型也是构造函数中的某个参数类型
  • index：
    • 用于指定注入的数据给构造函数中指定索引位置的参数赋值，索引从0开始
  • name：
    • 用于指定给构造函数中指定名称的参数赋值（常用）
  • value：
    • 用于提供基本类型和String类型的数据
  • ref：
    • 用于指定其他的bean类型数据。它指的是在IOC核心容器中出现的bean对象。无法注入集合
<bean id="UserDaoImpl" class="cn.jbit.news.dao.Impl.UserDaoImpl">
    <constructor-arg name=”age/构造函数中的数据名称” value=”赋值”></constructor-arg>
    <constructor-arg name=”Time” ref=”time”></constructor-arg>
</bean>
<bean id=”time” class=”java.util.Date”></bean>

优点

在获取bean对象时，注入数据时必须的操作，否则对象无法创建成功

缺点

改变了bean对象的实例化方式。如果在实例化时用不到这些参数也必须提供

注意：使用时需创建有参和无参构造器。如果constructor-arg没有指定name，则按有参构造器的参数顺序依次赋值

使用set方法

  1. 类里提供set方法

标签：property

位置：bean内

属性

  • name：
    • 类中的set方法去掉set后名称
  • value：
    • 用于提供基本类型和String类型的数据
  • ref：
    • 用于指定其他的bean类型数据。它指的是在IOC核心容器中出现的bean对象。无法注入集合
<bean id="UserDaoImpl" class="cn.jbit.news.dao.Impl.UserDaoImpl">
    <property name=”age/set方法中去掉set后的名称” value=”赋值”></property>
    <property name=”Time” ref=”time”></property>
    </bean>
<bean id=”time” class=”java.util.Date”></bean>

优势

创建对象时没有限制，可以使用默认构造函数

缺点

如果某个成员必须有值，则获取对象有可能set方法没有执行

p命名空间注入

属性：p

位置：bean的属性

用法：p:属性=”value”

<bean id="UserDaoImpl" class="cn.jbit.news.dao.Impl.UserDaoImpl" p:name="张三">

注意：

  1. 在beans标签里加入属性xmlns:p=”http://www.springframework.org/schema/p"
  2. 多个p直接用空格分隔
  3. 如果需要引用bean属性，则使用p:属性-rep=”bean属性的id”

注入复杂数据

  1. 使用set注入

位置：property内

用于给list集合结构注入的标签：list,array,set

用于给map集合结构注入的标签：map,props

结构相同，标签可以互换

<bean id="UserDaoImpl" class="cn.jbit.news.dao.Impl.UserDaoImpl">
    <property name=”集合的set名称”>
        <array>
            <value>aaa</value>
            <value>bbb</value>
            <value>ccc</value>
        </array>
    </property>
    <property name=”集合的set名称”>
        <list>
            <value>aaa</value>
            <value>bbb</value>
            <value>ccc</value>
        </list>
    </property>
    <property name=”集合的set名称”>
        <map>
            <entry key=”” value=””></entry>
        </map>
    </property>
</bean>
